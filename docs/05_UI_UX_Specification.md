# 05 UI/UX Specification

## UX direction

Hybrid: Mission Control command dashboard plus Notion-lite knowledge/evidence pages.

## Navigation

- Mission Control
- Projects
- Career
- Capabilities
- FGAs
- Evidence
- AI Workspace
- Knowledge Vault
- Decisions
- Weekly Review

## Layout

- Left sidebar for modules.
- Main workspace with cards and forms.
- Dashboard uses metric cards and operational lists.
- Evidence and Knowledge use document-like cards.

## User journeys

### Capture evidence

1. Open Evidence.
2. Add title, type, summary, tags, and optional file.
3. Link to capabilities/projects/tasks/FGAs/opportunities.
4. Generate or edit draft resume bullet and interview story.

### Weekly review

1. Open Weekly Review.
2. Review completed work, blockers, generated evidence, and recommendations.
3. Convert recommendations into new tasks or FGAs.

## Wireframe descriptions

### Mission Control

Top row metrics. Middle operational cards. Bottom recent decisions and evidence.

### Evidence page

Upload form on top. Existing evidence cards below. Each card shows source, tags, linked capabilities, resume bullet draft, and interview story draft.
